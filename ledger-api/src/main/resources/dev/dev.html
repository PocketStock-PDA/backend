5) 실시간 호가 (WebSocket / STOMP · UH1) 미연결
종목코드
STOMP /ws 연결 후 /topic/asking/{code} 구독 → LS 실시간(UH1) push 수신. 구독 시작 시 LS 등록, 해제 시 LS 해제(온디맨드).
6) 해외 실시간 호가 (KIS · HDFSASP0) 티커
상단 5번에서 먼저 STOMP 연결. 순수 티커(AAPL)만 보내면 서버가 거래소·세션(정규/주간)을 자동판별해 tr_key(DNAS…/RBAQ…)를 조립한다(#33) — 더이상 접두사 직접 안 붙임. 토픽 키도 안정적 티커(SYMB). 미국 정규장(한국 밤)·주간거래(10~16시) 시간에 데이터 수신.
7) 해외 실시간 체결가 (KIS · HDFSCNT0) 티커
상단 5번 연결 후 /topic/foreign/transaction/{티커} 구독 → 해외 실시간 체결가 push. 순수 티커(AAPL)로 구독(서버가 세션 자동판별 #33, 접두사 직접 안 붙임).
8) 국내 실시간 체결가 (LS · US3) 종목코드
상단 5번 연결 후 /topic/stock/trade/{code} 구독 → 국내 실시간 체결가(통합 체결, 현재가·등락·체결량) push.
9) 실시간 환율 USD/KRW (LS · CUR)
상단 5번 연결 후 /topic/currency/usd-krw 구독 → LS 실시간(CUR) 현물USD 환율 push. 종목코드 없는 단일 토픽. 외환시장 개장시간(평일 주간)에 데이터 수신.
10) 환율 조회 (REST · /api/exchange/rate) 
결과 raw JSON
캐시(LS CUR SSOT)의 기준율 + 매수/매도 적용환율(스프레드 0.96%·우대 90% 내재). 첫 틱 수신 전(콜드스타트)엔 502. 상시구독이라 9번 구독 없이도 값이 쌓임.
11) 환전 이력 (REST · /api/exchange/history)
fx_transactions 최신순. 환전 체결(/krw-to-usd)은 CMA 연동 후라 그 전까진 비어 있음.

13) 환전 체결 (REST · /api/exchange/krw-to-usd · /usd-to-krw) 방향 금액 비번 
결과 raw JSON
캐시(LS CUR)에 환율이 쌓인 뒤(콜드스타트 시 502) 체결. CMA 풀은 실물 cma_balances(#56 실 어댑터, 스텁 제거 #57) — `/api/cma/account` 개설 + `/api/cma/collect`로 원화 적립 후 환전. 체결 성공 시 11번 이력에 행이 쌓임. 원화→달러 환전분은 CMA USD풀에 적립되고, `/api/cma/transfer {market:OVERSEAS}`로 해외 위탁 USD 예수금까지 충전된다.
